# 角色视角与权限模拟说明

## 目标

第三阶段先实现演示级角色视角，让同一套数据在不同角色下呈现不同页面入口和不同人员范围，为后续真实登录与权限系统预留结构。

当前实现是“权限模拟”，不等同于生产环境鉴权。用户可在 Streamlit 侧边栏切换角色，便于演示 HR、导师、实习生、招聘同学的不同视角。

## 角色与页面权限

| 页面 | HR | 导师 | 实习生 | 招聘同学 |
| --- | --- | --- | --- | --- |
| 首页 | 可访问 | 可访问 | 可访问 | 可访问 |
| HR Dashboard | 可访问 | 不可访问 | 不可访问 | 可访问 |
| Intern Profile | 可访问 | 可访问 | 可访问 | 可访问 |
| Mentor Assistant | 可访问 | 可访问 | 不可访问 | 不可访问 |
| AI Weekly Report | 可访问 | 不可访问 | 不可访问 | 可访问 |
| Intern Growth Assistant | 可访问 | 可访问 | 可访问 | 不可访问 |

## 数据可见范围

- HR：可查看全部实习生数据。
- 招聘同学：可查看全局适岗与风险相关数据，用于招聘和培养复盘。
- 导师：选择一个模拟导师账号后，仅查看该导师带教的实习生。
- 实习生：选择一个模拟实习生账号后，仅查看本人数据。

## 实现位置

- `services/auth_service.py`：角色定义、权限矩阵、范围过滤、页面侧边栏控件。
- `pages/1_HR_Dashboard.py`：限制为 HR / 招聘同学访问。
- `pages/2_Intern_Profile.py`：按角色控制可选实习生范围。
- `pages/3_Mentor_Assistant.py`：限制为 HR / 导师访问，导师仅看带教范围。
- `pages/4_AI_Weekly_Report.py`：限制为 HR / 招聘同学访问。
- `pages/5_Intern_Growth_Assistant.py`：限制为 HR / 导师 / 实习生访问，并按角色缩小范围。
