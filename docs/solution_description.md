# 方案说明

本项目设计“实习能量站：AI 驱动的业务部实习生成长导航智能看板”，用于解决业务部校招实习生带教过程中的三类问题：导师带教依赖个人经验，缺少统一节奏；实习生不知道每阶段该学什么、做什么；HR 和招聘同学难以及时掌握整体适岗情况。

方案面向 HR、业务导师、实习生和招聘团队四类角色，设计了五个核心页面：HR 总览看板、实习生个人画像、导师带教助手、实习生成长助手和 AI 周报生成。HR 看板展示 20 名实习生的岗位分布、平均任务分、平均适岗分、风险等级和适岗排名；个人画像展示单个实习生的基础信息、能力评分、导师反馈、任务记录、风险原因和行动建议；导师带教助手将自然语言反馈结构化为优点、问题、风险等级、下周带教动作和反馈话术；实习生成长助手根据岗位、周次和能力短板生成学习重点、推荐任务、交付物和向导师确认的问题；AI 周报模块将群体数据整理成 HR 可直接复用的周报正文。

AI 工具选型上，本 MVP 采用“规则评分 + 模板生成”的稳定方案，并预留 Prompt 模板用于后续接入大模型 API。这样既能保证无网络或无 API Key 时 Demo 可稳定运行，也能体现 AI 在 HR 场景中的核心价值：把分散数据和自然语言反馈转化为可解释结论和可执行动作。适岗评分采用五个维度：任务完成度 30%、导师评价 25%、学习主动性 20%、沟通协作 15%、岗位技能匹配 10%。风险识别结合适岗分、任务分和导师反馈关键词，输出低风险、需关注和高风险。

项目迭代分为五步：第一步完成工程骨架和 20 名实习生模拟数据；第二步沉淀评分与风险规则；第三步实现 HR 看板和个人画像；第四步加入导师反馈分析和 AI 周报；第五步反思 MVP 缺口并补齐实习生成长助手。效果评估可通过任务完成率、导师反馈覆盖率、风险识别数量、AI 建议可采纳性和 HR 查看效率进行衡量。

当前 Demo 已形成“整体态势 -> 风险定位 -> 个人画像 -> 反馈分析 -> 成长建议 -> 周报生成”的完整闭环，可用于课程作业展示和后续部署。
